### - [ ] 2. 收紧类型安全（244 处 any）  📊 已扫 hotspots
**Hotspot 分析（`as any` top）**：
- `crdt/YDoc.ts` 19（yjs interop，多数合理）
- `components/PathEditor.vue` 19（SVG 路径类型，复杂）
- `components/EditorLayout.vue` 10（layout DOM 体操）
- `plugins/typography.ts` 9（文本编辑内部）
- `mixins/0.context.ts` 8（editor 扩展类型展宽，设计如此）

**判断**：top 5 hotspot 各自都需要专项理解（yjs / SVG / 文本编辑器内部），surgical 清理时间成本高、回归风险有。**建议先在低风险文件（如 `autoNest.ts` 3 处、`Overlay.vue` 4 处）做样板治理 + 沉淀模式**，再批量。

**待决策**：目标范围（清零 / 砍 50% / 仅治理 `as any`），是否加 ESLint rule 禁新增。

**现状（原始）**：`packages/mce/src` 出现 `: any` 106 次、`as any` 138 次。0 处 `@ts-ignore`/`@ts-expect-error`。根 tsconfig 开启了 `strict: true` + `noImplicitAny: true`，但被显式 any 绕过。

**风险**：插件间事件 ctx、Node↔Element2D 互转、外部库 schema 接入处类型保障失效。重构时易出隐藏回归。

**建议**：
- 先扫出 top 10 文件 + 顶部聚集点（哪里 any 最密集）
- 做"语义化窄类型"替换：插件事件 ctx 用 union、Node 守卫用类型谓词
- `as any` 比 `: any` 更隐蔽，优先治理
- 不强求清零，**目标砍 30~50%**

**Effort**：分批，1 周内能砍 30%。

### - [ ] 3. 构建产物 minify + 分 chunk  🚧 minify 已开
**已完成**：`vite.config.ts` `minify: false` → `'esbuild'`，dist/index.js **614 KB → 476 KB（22% ↓）**，gzip 151 → 129 KB。

**待续**：chunk 拆分等 P1.5 按需注册落地后一并规划。

**现状（原始）**：`packages/mce/dist/index.js` 614 KB 单 bundle，`vite.config.ts` 显式 `minify: false`，无 chunk 设计；rollupOptions external 已含 dependencies/peerDependencies。

**风险**：依赖方（用 mce 的应用）首屏负担大、tree-shake 失效。

**建议**：
- 开 `minify: 'esbuild'`（预计砍 30~40%）
- 按 plugins/exporters/crdt 切多个 chunk
- 评估 external 完整性：确认 vue/yjs/y-protocols 也已 external
- 配合 P1.5（按需注册）效果倍增

**Effort**：1 天。

### - [x] 12. i18n 扩展性  ✅ 已支持
**核实结论**：`mixins/0.locale.ts` 已经支持三种运行时扩展方式：
- `editor.registerMessages({ en: {...}, zhHans: {...} })`——插件追加自己的 i18n
- `options.locale.messages = { ja: {...}, ... }`——构造时注入任意新语言
- `options.t = (key, fallback) => ...`——完全自定义翻译函数（最高优先级）

`messages` computed 用 `lodash-es` `merge` 把核心 locale + 插件 messages + 自定义 messages 三层合并，符合"可外部扩展"诉求。当前 2 语言只是默认值，不是限制。

**待续**：若海外/多团队要求扩展，在 README 加段说明三种方式即可。本条架构上无负债。
